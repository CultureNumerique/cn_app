1. Avant-Propos

1-1. Cours

Ce module présente un tutoriel de prise en main de la chaîne éditoriale d'un cours sur Esc@pad. Nous vous présenterons les bases pour écrire votre cours en Markdown et en Gift . Pour plus d'informations sur ces 2 langages, veuillez vous référez au lien suivant:

  • Markdown
  • Gift

2. Markdown

2-1. Introduction

Markdown est un langage très simple pour écrire du HTML de manière détournée. Vous pouvez voir sur votre droite (pensez à affichez l'aperçu ci-dessus !) un aperçu de la syntaxe Markdown une fois interprétée. Vous pouvez vous entraîner sur ce cours dans la partie Entraînement située à la fin de ce cours.

2-2. La syntaxe

Nous allons maintenant débuter une rapide revue des élements de la syntaxe Markdown.

Paragraphe

Pour créer plusieurs paragraphes, il suffit de séparer les deux zones par une ligne vide.

Exemple 

Je suis un premier paragraphe (je laisse ensuite une ligne vide).

Je suis un second paragraphe.

Résultat

Je suis un premier paragraphe (je laisse ensuite une ligne vide).

Je suis un second paragraphe.

Attention ! Vous ne pouvez malheureusement pas sauté de ligne au sein d'un même paragraphe.

Mise en valeur

Pour mettre en valeur une portion de votre texte, il suffit d'entourer les mots de votre choix entre des étoiles * ou des traits de soulignement (underscore) _ . Vous pouvez soit mettre votre texte en italic , soit en gras .

Exemple

Pour mettre en italic, vous pouvez: 

Mettre votre *texte* ainsi ou comme _ceci_

Pour mettre en gras : Mettre votre texte ainsi ou comme ceci

Résultat

Mettre votre texte ainsi ou comme ceci

Mettre votre texte ainsi ou comme ceci

Les titres

Pour créer vos titres sur Esc@pad il vous suffit de mettre un ou plusieurs # suivant le niveau de votre titre: 

# Titre de niveau 1
## Titre de niveau 2
### Titre de niveau 3

Attention

Sur Esc@pad, votre document doit OBLIGATOIREMENT commencé par un titre de niveau 1 sans quoi, votre texte ne sera pas pris en compte. Le texte situé en dessous d'un titre de niveau 1 est placé obligatoire dans une partie Cours si vous ne précisez pas de titre de niveau 2

Exemple 

# Titre de niveau 1
## Titre de niveau 2
### Titre de niveau 3
Je suis un contenu

# Titre de niveau 1
Je suis un contenu automatiquement placé dans un **Cours**
### Titre de niveau 3

Résultat

Pour afficher le résultat, enlevez le \ devant chaque #

# Titre de niveau 1 ## Titre de niveau 2 ### Titre de niveau 3 Je suis un contenu # Titre de niveau 1 Je suis un contenu automatiquement placé dans un Cours ### Titre de niveau 3

Les listes

Il existe deux types de listes gérés par Markdown :

Listes à puces

Exemple 

* Une puce
* Une autre puce
* Et une dernière puce

Résultat

  • Une puce
  • Une autre puce
  • Et une dernière puce

Listes numérotés

Exemple 

1. Première partie
2. Deuxième partie
3. Troisième partie

Résultat

  1. Première partie
  2. Deuxième partie
  3. Troisième partie

Les citations

Fonctionne comme pour les e-mails avec > ou » .

Exemple 

> Texte cité. Pour chaque
> nouvelle ligne il faut rajouter un
> chevron devant

Résultat

Texte cité. Pour chaque nouvelle ligne il faut rajouter un chevron devant

Code informatique

Pour insérer du code source, il suffit de laisser un espace à la fin du paragraphe et d'indenter (en ajoutant 4 espaces ou une tabulation) avant d'écrire son texte. 

def algo (i) :
    i = i + 1
    return i

Code en ligne

Pour faire ressortir un morceau de code au milieu d'un paragraphe, il suffit d'entourer la portion de texte d'accent grave `

Exemple 

L'agorithme précédent permet d'augmenter `i` de 1.

Résultat

L'agorithme précédent permet d'augmenter i de 1.

Caractère non interprété

Pour que Markdown n'interprète pas certains symboles il suffit d'ajouter \ devant le symbole en question 

\` \* \** \{texte}

` * ** {texte}

Les liens

Pour ajouter un lien dans votre fichier, vous devez placer le texte du lien entre crochets suivis de l'URL entre parenthèses [nom_affiché](http:adresse_du_site) :

Exemple 

[Mon lien](https://culturenumerique.univ-lille3.fr/index.html)

Résultat

Mon lien

Info Avec Esc@pad les liens s'ouvrent dans une nouvelle fenêtre.

Les images

Les images en Markdown s'ajoutent de la manière suivante:

  • Lien vers une image hébergée sur internet 
![Mon image](https://culturenumerique.univ-lille3.fr/static/img/logo_plat_vert.svg)
  • Lien vers un media : Pour ajouter des images de votre cours depuis votre ordinateur, vous devez l'ajouter comme ceci. 
![Mon image](base/module/media/Logo_plat_vert.svg)

Pour l'instant, une image par défault est affichée mais lorsque vous générerez votre cours (en suivant bien l'indication de la petite bulle d'aide avant de créer votre cours) votre image sera remplacée.

Attention

Évitez au maximum de mettre des accents dans le nom du fichier ou des majuscules dans l'extension de l'image.

Les vidéos

Pour l'instant, Esc@pad ne peut intégrer que les vidéos venant de Vimeo . Pour intégrer directement votre vidéo dans votre cours il suffit de {: .cours_video} .

Exemple 

[Nom de video]( https://vimeo.com/93350435 ){: .cours_video }

Attention ! Pas de prévisualisation pour les vidéos.

3. Les activités

3-1. Cours

Les activités regroupent les questions de votre module. Les questions sont rédigées en format Gift, elles seront interprétées par Esc@pad.Il existe 3 types d'activités :

  • Les Comprehensions : on y met généralement des questions très simple de cours sous forme de QCM.
  • Les Activités : [à compléter]
  • Les Activités-Avancées : [à compléter]

3-2. Intégrer une activité dans votre cours

Les sous-sections contenant ces activités sont elles délimitées par 2 séries de ``` placées en début de ligne et dont la première est suivie du type d'activité. 

```comprehension ou active ou activite-avancee
[questions]
```

3-3. Syntaxe Gift

(documentation très inspirée de celle de moodle )

Memento Gift

Voici un tableau pour les différents symboles utilisés dans Gift.

Symbole Utilité
::Titre:: Titre de la question (fortement conseillé)
[...Format...] Format du passage suivant de texte ( markdown fortement conseillé)
{ Début des réponses
{T} ou {F} Réponses vrai ou faux; aussi
{ ... =Bonne réponse ... } Bonne réponse pour les choix multiples ou les textes troués
{ ... ~Mauvaise réponse ... } Mauvaise réponse pour les choix multiples ou les textes troués
{ ... =Élément -> réponse ... } Réponse pour les questions d'appariement
#FeedbackLocal Feedback pour une réponse
####FeedbackGeneral Feedback pour la question
{# Début des questions numérique
Réponse:marge d'erreur Réponse numérique acceptée à l'intérieur d'une marge d'erreur
Limite inférieure..Limite supérieure marges de valeurs inférieure et supérieure acceptées dans une question à réponse numérique
=%n%réponse:marge d'erreur n pour cent accordé à une réponse numérique acceptée à l'intérieur d'une marge d'erreur
} Fin des réponses
\caractère Une barre oblique inverse permet de bloquer la signification de ~, =, #,
\n Ajout un saut de ligne dans le texte de la question -- les lignes vides délimitent les questions

3-4. Les questions

Format d'une question

Le format d'une question en GIFT est comme-suit: 

::Titre::
Texte de la question
{les réponses
####Feedback général}

::Titre::
Texte de la question
{réponses}

Pour certains types de questions, il est possible d'ajouter des feedbacks locaux pour chaque réponse.

Réponse unique

Une seule réponse est valide parmis les propositions.

Exemple très simple de réponse unique: 

Qui repose dans la Grant's tomb ? {=Grant ~Personne ~Napoléon ~Churchill ~Mère Teresa}

Exemple avec plus d'éléments de la syntaxe GIFT: 

// Question : 1 Nom : Grant's tomb
::Grant's tomb::Qui repose dans la Grant's tomb à New-York ? {
=Grant
~Personne #C'était vrai pendant 12 ans, mais la dépouille de Grant a été enterrée dans cette tombe en 1897.
~Napoléon #Il a été enterré en France.
~Churchill #Il a été enterré en Angleterre.
~Mère Teresa #Elle a été enterrée en Inde. }

Choix multiple avec plusieurs bonnes réponses

Plusieurs réponses vraies parmis un choix de réponses. Le pourcentage permet d'attribuer le bon nombre de points suivant la réponse donnée par l'utilisateur:

  • ~%-100% : Réponse fausse
  • ~%-50% : La moitié des points 
Parmi ces personnes, nommez-en deux qui sont enterrées dans la Grant's tomb. {   ~%-100%Personne   ~%50%Grant   ~%50%L'épouse de Grant   ~%-100%Le père de Grant }

Précisons que si le pourcentage n'est pas un nombre entier (par exemple pour 3 bonnes réponses) il faut écrire le nombre suivi de 5 chiffres après la virgule (le séparateur utilisé doit être le "point" et non la "virgule"). 

question {
~%33.33333%Bonne réponse
~%33.33333%Bonne réponse
 ~%33.33333%Bonne réponse
~%0%Mauvaise réponse
~%0%Mauvaise réponse }

Vrai / Faux 

::Q1:: 1+1=2 {T}
::Q2:: 1+1=3 {FALSE#Feedback si l'utilisateur s'est trompé#Feedback si l'utilisateur a bon}

Réponse courte

Les réponses d'une question de type réponse courte sont toutes précédées d'un symbole d'égalité (=), indiquant que toutes ces réponses sont bonnes. 

Deux plus deux égalent {=quatre =4 =IV}

Appariement

Les différents appariements commencent par un symbole d'égalité (=) et sont séparés par ce symbole « -> ». Il doit y avoir au moins trois appariements (au moins trois éléments différents d'un côté et au moins deux éléments différents de l'autre). 

Appariez les pays suivants avec les capitales correspondantes. {
=Canada -> Ottawa
=Italie -> Rome
=Japon -> Tokyo
=Inde -> New Delhi }

Mot manquant

Le format mot manquant insère automatiquement un trou au milieu d'une phrase. Pour utiliser le format mot manquant, mettez les réponses où vous voulez qu'un trou se crée dans le texte. 

Il faut débourser { =rien} pour utiliser Esc@pad.

Attention ! On ne peut pas mettre plusieurs réponses à trou dans un seul texte, il faut pour cela créer une nouvelle question à chaque fois.

Question à réponse numérique

La section réponse pour les questions à réponse numérique doivent commencer par un dièse ( # ). Les réponses numériques peuvent inclure une marge d'erreur, qui doit être mentionnée après le bonne réponse, et être précédée d'un deux-points ( : ). Ainsi, par exemple, si la bonne réponse peut être n'importe quel nombre entre 1,5 et 2,5, alors elle sera écrite comme { #2:0,5 }. Ceci indique que 2 avec une marge d'erreur de 0,5 est la bonne réponse (c'est-à-dire l'empan entre 1,5 et 2,5). Si aucune marge d'erreur n'est mentionnée, elle sera de zéro.

Voici un simple format de question à réponse numérique. Elle tolérera une marge de 5 ans. 

En quelle année Ulysses S. Grant est-il né ? {#1822:5}

C'est une bonne idée de vérifier les limites de la marge. Par exemple, dans le prochain exemple, 3,141 n'est pas une bonne réponse et 3,142 est une bonne réponse selon la marge. 

Quelle est la valeur de pi (incluant 3 chiffres après la virgule) ? {#3,14159:0,0005}.

Optionnellement, les réponses numériques peuvent être rédigées en tant qu'étendue comme dans cet exemple : {#valeur minimale..valeur maximale}. 

Quelle est la valeur de pi (incluant 3 chiffres après la virgule) ? {#3,141..3,142}.

Attention ! Esc@pad ne gère par les listes de réponses numérique.

Composition

Une question de type composition est tout simplement une question avec un champ de réponse vide. Les accolades indiquant les réponses doivent rester vides, puisque ce type de question ne peut pas être corrigé automatiquement. 

Écrivez une courte biographie de Dag Hammarskjöld. {}

Description

Une question de type Description n'a aucune réponse. Ce type de question ne sert qu'à insérer du texte fixe comme une consigne dans votre test. 

Vous pouvez utiliser un crayon et un papier pour les questions d'appariement suivantes.
